## Principle

The postulant portal is a tracking interface. It must be simpler than the internal admin UI.

## Layout rules

- One primary action per screen.
- Avoid long mixed-content pages.
- Group content by user intent: status, documents, payments, meetings, messages, certificate.
- Show internal process only as simplified milestones.

## Home screen should answer

1. What is my current dossier?
2. What is the current status?
3. Is an action expected from me?
4. What changed recently?
5. Where do I click to see details?

## Dossier screen should answer

1. What phase/status is my dossier in?
2. Which documents are valid, in review, or missing?
3. Are there payments expected?
4. Are meetings scheduled or completed?
5. What notifications/decisions were published?
6. What is the certificate status?
